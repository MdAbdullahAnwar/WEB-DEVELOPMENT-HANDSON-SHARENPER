Ques1. What is the "temporal dead zone" in JavaScript?

Ans: It refers to a period of time when a variable is accessible but not defined.



Ques2. Const person= { Name:p1, Age:24 } Can we change age to 25? Can we do person.push(age:25);

Ans: Yes, we can change person.Age = 25; — that’s perfectly legal.
No, we cannot use person.push(age:25) because person is not an array.

Ques4. What is the main difference between setTimeout, setInterval, and setImmediate in JavaScript?

Ans: setTimeout executes code once after a specified delay, setInterval repeatedly executes code at a specified interval, and setImmediate executes code immediately after the current event loop.



Ques5. What is the main difference between null and undefined in JavaScript?

Ans: null represents an explicitly assigned "empty" or "non-existent" value, while undefined represents an uninitialized or missing value.



Ques6. What is the main difference between "undeclared" and "undefined" variables in JavaScript?

Ans: "Undeclared" variables are not a valid concept in JavaScript.
